Procedural and Object-Oriented Programming; Classes; Working with Instances; and Techniques for Designing Classes

Procedural Programming

Writing programs made of functions that perform specific tasks
procedures typically operate on data items that are separate from the procedure
data items are commonly passed from one procedure to the another
Focus: - to create procedures that operate on the program's data

Object-Oriented Programming

Focused on creating objects -- an Object is an entity that contains data and procedures
Data is known as 'Data Attributes' and Procedures are known as 'Methods'

Encapsulation is combining data and code into a single object
a List is an object - it will contain a sequence of data - but if we create a List named L and dir(L) - we will see the methods available to use on the object 'L' (List)

ex: insect_class.py

Data Hiding

(Getters and Setters)- object's data attributes are hidden from code outside the object.
this keeps Access restricted to the object's methods, which protects from accidental corruption. (Outside code does not need to know internal structure of the object)

Object Reusability - same object can be used in multiple programs.
  • Data Attributes - define the state of an object; ex: clock object will have second, minute, hour data attributes
  • Public Methods - allow external code to manipulate the object; ex: set_time, set_alarm_time
  • Private Methods - used for objects inner workings; ex: circuit board, battery, wiring

Classes

  • Class - code that specifies the data attributes and methods of a a particular type of object. ex: blueprint of a house, cookie cutter
  • Instance - object created from a class; ex: houses built from the blueprint, or cookies made with that cutter. There may be many instances of one class
the Class is an abstraction and the Instance is a specific object that can be acted upon the Class Definitions are a set of statements that define a class' methods and data attributes

Format: begin with class Class_name: -- class names often start with an Uppercase letter
Method definition like any other python function definition
self parameter: required in every method in the class.
self references the specific object that the method is working on; some programmers use 'x' instead of 'self'

Initializer Method automatically executed when an instance of the class is created
def __init__(self): - Initializes object's data attribures and assigns 'self' parameter to the object that was just created; it is Usually the first method in a class definition
all private methods have the double underscore "__"

To create a new instance of a class call the initializer method - My_instance = Class_Name()

To call any of the class methods using the created instance, use 'dot notation'
My_instance.method(); because the 'self' parameter references the specific instance of the object, the method affect this instance;
Reference to 'self' is passed automatically
