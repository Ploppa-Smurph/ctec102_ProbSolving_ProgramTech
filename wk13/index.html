class Class_name: is the format for creating a class.
the self parameter must be included in every method in a class. 'Self' references the specific object instance that the method is working on.
def __init__(self): - the Initializer Method is automatically executed when an instance of the class is created.
the initializer method is usually the 1st method in the class, initializes the object's data attributes and assigns self parameter to the object that was created.

the 'String' Method: __str__ displays an object's state.
the string method is automatically called when the object is passed as an argument to the print function or the str function.

Getter is the Accessor method: it returns a value from a classes attribute without changing it
Setter is the Mutator method: stores or changes the value of a data attribute

UML Diagram (Unified Modeling Language)- standard diagrams for graphically depicting object-oriented systems
UML is a box divided into 3 sections
  • Top Section: name of the class
  • Middle Section: list of data attributes
  • Bottom Section: list of class methods

1st thing you want to do when developing a program is to come up with the Classes.
you identify the real-world problems.
  1. Get written description of the problem
  2. Identify all nouns in the description, each of which is a potential class.
  3. Refine the list to include only classes that are relevant to the problem
A lot of times the verbs that are used with the nouns are how you will name the methods in the class.
A Classes Responsibilities - often this will help identify the data attributes of the class.
the Classes's Actions are it's methods == what it is in charge of doing.
