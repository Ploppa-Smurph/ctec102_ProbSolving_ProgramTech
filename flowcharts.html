some Flowchart and Pseudocode notes

the rounded rectangles are the 'start' and 'end'
parallelogram is either an 'input' or 'output' statement
rhombus is a conditional statement and can show up inside loops or many other places
the rectangle - the verb- an operation occurs, a variable is set, a calculation is made, etc... but an operation happens: normally using an assignment operator

another set of notes

schematic that uses special images to represent an algorithm or process.

rounded rectangle - start; not all flow charts will have an end
rectangle - the action verb, ex: add 1, turn on motor, turn off light, rotate part...
parallelogram - input / output - indicates a manual operation is needed, ex: type in weight, check the balance, time the operation...
rhombus / aka diamond - Decision, has an outcome of 'Yes' or 'No'; ex: is > 10; does the weight meet specifications, has the count been reached?
arrows - the arrows indicate the flow of the chart. only 1 arrow can represent 'outputs'; multiple arrows may represent inputs

logic of flowchart

what do we want the outcome to be? and how will we start? always start with 'start'
